# Feature Task Prompt Template

Read:
- `AGENTS.md`
- the task file,
- relevant screen spec,
- relevant component contracts,
- relevant architecture/design docs.

Do not code immediately.

First produce:
1. files you will inspect,
2. implementation plan,
3. files you expect to change,
4. tests you will add/update,
5. risks or ambiguities.

After approval, implement the smallest change that satisfies the task.

Before final response:
- run all required commands,
- save screenshots/artifacts if UI changed,
- fill out `tasks/FINAL_REPORT_TEMPLATE.md`,
- mark Status as PASS only if all gates passed.

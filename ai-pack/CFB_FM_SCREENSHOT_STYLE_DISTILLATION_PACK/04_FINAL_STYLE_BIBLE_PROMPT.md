# Final Style Bible Prompt

Use this after you have several **chunk summaries**.

Paste the chunk summaries and then use this prompt.

---

You are creating a final UI art-direction and implementation guide for a fictional college football management game.

You have multiple chunk summaries derived from many analyzed screenshots.

Your job is to synthesize them into a **single final style bible**.

## Your tasks

Create a polished markdown document with the following sections:

# CFB-FM UI Style Bible

## 1. Visual North Star
Describe the target vibe in plain language.

## 2. Core Identity
Describe:
- what kind of game UI this is
- what it should feel like
- what it should not feel like

## 3. Primary Screen Categories
List the main categories of screens the game uses.

## 4. App Shell Rules
Describe:
- top bar
- main navigation
- sub-navigation
- main workspace
- side rails
- footer / bottom action area
- continue/action zone

## 5. Layout System
Describe:
- page templates
- card layouts
- dashboard rules
- table/list rules
- detail/profile rules
- modal rules
- spacing and density rules

## 6. Component Library
Define reusable components:
- nav tabs
- section headers
- stat cards
- hero image module
- data grid/table
- standings panel
- schedule panel
- profile card
- CTA buttons
- utility/status bar
- ticker
- badges
- ratings
- etc.

## 7. Visual Design Rules
Describe:
- color palette
- surfaces
- accent usage
- status color usage
- borders/dividers
- textures
- shadows
- depth

## 8. Typography Rules
Describe:
- headline style
- section labels
- body text
- numeric/stat styling
- button styling

## 9. Information Hierarchy Rules
Describe what should be primary, secondary, tertiary.

## 10. Professional Polish Rules
List concrete signals that make the UI feel commercially credible.

## 11. Anti-Patterns
List what makes a UI feel amateur and must be avoided.

## 12. Image-Generation Prompt Template
Write a prompt template for generating new screenshots in this style.

## 13. Negative Prompt
Write a good negative prompt for avoiding amateur/incorrect outputs.

## 14. Design Tokens
Suggest useful tokens:
- colors
- spacing scale
- border radius
- line opacity
- typography sizes
- panel padding
- icon sizing

## 15. Screen Template Examples
Give specific template guidance for:
- title screen
- dashboard
- recruiting screen
- roster screen
- player profile
- schedule screen
- staff screen
- news/program desk
- postgame report

## 16. Final Summary
One strong summary paragraph.

## Hard rules

- Write decisively.
- Be concrete.
- Optimize for implementation.
- Optimize for both UI coding and image generation.
- Preserve the fact that these are **video game screens**.
